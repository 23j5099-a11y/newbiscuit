# ビスケット融合システム 仕様書

## 1. 目的

本システムは、ビスケットプログラムの基本仕様を土台にしつつ、現在開発中のシステムが持つ「作品保存」と「幼児向けの自由制作」の考え方を取り入れた、幼児向けビジュアルプログラミング環境である。

絵を使って、メガネによる変化のルールを作り、そのルールが繰り返し働くことで、動き続ける作品を作れるようにする。

## 2. 採用方針

### 2.1 ビスケットプログラムの仕様を利用する部分

以下はビスケットプログラムの仕様を基本仕様として利用する。

| 項目 | 採用する仕様 |
|---|---|
| 共通する基本部分 | 絵、ステージ、メガネ、変化、繰り返し |
| 実行のされ方 | メガネのルールが自動的に繰り返し適用される |
| メガネの判定 | メガネ左側の絵と配置を、ステージ上の状態から探す |
| 複数の絵の扱い | メガネ左側に複数の絵を置き、配置関係を条件として扱う |
| 動きの意味 | 左右の絵の位置差によって移動や変化を表す |
| ランダム性 | 同じ条件に複数のメガネがある場合、どれかが選ばれる |
| 画面構成 | 部品置き場、ステージ、メガネ置き場を中心に構成する |

### 2.2 今のシステムの仕様を利用する部分

以下は現在開発中のシステムの仕様を利用する。

| 項目 | 採用する仕様 |
|---|---|
| 作品保存 | 作った作品やログをサーバー側に保存する |
| 幼児向けの自由制作 | 用意された画像を選び、ドラッグして作品を作れるようにする |

## 3. 基本概念

### 3.1 絵

絵は、作品を構成する最小単位である。

絵は次の場所で使われる。

- 部品置き場
- ステージ
- メガネの左側
- メガネの右側

同じ絵を複数個使うことができる。

### 3.2 部品置き場

部品置き場には、作品に使える絵が表示される。

幼児が自分で描画する代わりに、現在のシステムと同じく `public/images/` に用意された画像を部品として表示する。

### 3.3 ステージ

ステージは、作品が実際に動く場所である。

部品置き場から絵をドラッグしてステージに置くことができる。ステージ上の絵は、メガネのルールによって移動したり、別の絵に変化したりする。

### 3.4 メガネ

メガネは、変化のルールを表す。

メガネは左側と右側に分かれる。

| 場所 | 意味 |
|---|---|
| 左側 | 変化前の状態、探す条件 |
| 右側 | 変化後の状態 |

メガネの左側に置かれた絵の配置がステージ上に見つかると、その部分が右側の配置へ変化する。

## 4. 画面構成

画面はビスケットプログラムの仕様を基本に、次の3領域で構成する。

| 領域 | 役割 |
|---|---|
| 部品置き場 | 使える絵を一覧表示する |
| ステージ | 絵を置き、作品が動く場所 |
| メガネ置き場 | メガネを作り、変化ルールを置く場所 |

現在のシステムのように説明的な3パネル構成にしてもよいが、仕様上はビスケットプログラムと同じく、部品置き場・ステージ・メガネ置き場を基本単位とする。

## 5. 実行のされ方

実行のされ方は、ビスケットプログラムの仕様を利用する。

### 5.1 常に変化が続く

ステージ上の絵は、メガネの条件に合うたびに変化する。

1回実行して終わるのではなく、メガネのルールが繰り返し適用される。



## 6. メガネの判定

メガネの判定は、ビスケットプログラムの仕様を利用する。

### 6.1 左側を条件として探す

メガネの左側は、ステージ上から探す条件である。

左側に絵が1つだけある場合は、ステージ上の同じ絵を探す。

左側に複数の絵がある場合は、絵の種類だけでなく、それらの配置関係も条件として扱う。

### 6.2 右側へ変化する

条件に合う部分が見つかったら、対応する部分をメガネの右側の状態へ変化させる。

右側に違う絵が置かれていれば、絵が変化する。

右側で位置が変わっていれば、絵が移動する。

右側で絵が増えていれば、絵が増える。

右側で絵が消えていれば、絵が消える。


### 7.1 配置全体をルールにする

メガネの左側に複数の絵を置いた場合、それらは別々の条件ではなく、ひとまとまりの条件として扱う。

例えば、左側に「卵」と「かえる」が並んでいる場合、ステージ上でも同じような位置関係にある「卵」と「かえる」を探す。

### 7.2 右側の配置へ変わる

左側と右側の対応によって、複数の絵が同時に動いたり、変化したりする。

これにより、単なる画像の置き換えではなく、複数の絵の関係を使ったプログラムを作ることができる。



### 8.1 左右の位置差が動きになる

メガネの左側と右側で同じ絵の位置がずれている場合、そのずれがステージ上の移動になる。

例:

```text
左側: かえるが中央にある
右側: かえるが少し右にある
```

このメガネは「かえるが右へ動く」という意味になる。

### 8.2 絵の違いが変化になる

メガネの左側と右側で絵が違う場合、左側の絵が右側の絵へ変化する。

例:

```text
左側: おたまじゃくし
右側: かえる
```

このメガネは「おたまじゃくしがかえるになる」という意味になる。

### 8.3 移動と変化は同時に表現できる

右側で絵が変わり、位置も変わっている場合、絵は移動しながら変化する。

## 9. ランダム性

ランダム性は、ビスケットプログラムの仕様を利用する。

同じ絵や同じ条件に対して複数のメガネがある場合、どのメガネが適用されるかは1つに固定しない。

例えば、次の2つのメガネがある場合:

```text
かえる -> 右へ動くかえる
かえる -> 下へ動くかえる
```

ステージ上のかえるは、右に動くこともあれば、下に動くこともある。

このランダム性により、自然な動きや予想しにくい作品を作ることができる。

## 10. 作品保存

作品保存は、現在のシステムの仕様を利用する。

### 10.1 保存対象

次の情報を作品データとして保存する。

| 対象 | 内容 |
|---|---|
| ステージ | 置かれている絵、位置、種類 |
| メガネ | 左側と右側に置かれた絵、位置、種類 |
| 部品 | 使用可能な画像の情報 |
| 作成日時 | 作品が保存された日時 |

### 10.2 保存形式

作品はJSON形式で保存する。

現在のシステムと同じく、サーバー側で保存・読み込みできる構成にする。

### 10.3 保存タイミング

保存は次のタイミングで行う。

- 作品を保存したとき
- 必要に応じて自動保存するとき

実行中の一瞬ごとの状態をすべて保存するのではなく、作品として再現できる状態を保存する。



### 11.1 用意された絵を使う

幼児が最初から絵を描かなくても作品を作れるように、あらかじめ用意された画像を部品置き場に表示する。

画像は `public/images/` 以下に置く。

対応する画像形式:

- `.png`
- `.jpg`
- `.jpeg`
- `.gif`
- `.svg`
- `.webp`

### 11.2 ドラッグ操作を基本にする

幼児が操作しやすいように、基本操作はドラッグ&ドロップにする。

| 操作 | 内容 |
|---|---|
| 部品をステージへドラッグ | ステージに絵を置く |
| 部品をメガネへドラッグ | 変化ルールを作る |
| ステージ上の絵をドラッグ | 絵の位置を変える |
| メガネ内の絵をドラッグ | ルール内の位置関係を変える |

### 11.3 自由制作を妨げない



メガネの数、ステージ上の絵の数、使う画像の種類に上限を設けないことを基本とする。

## 12. ログ

ログは、作品保存を補助する情報として扱う。

現在のシステムでは、メガネに何の絵を入れたか、入れ替えたか、消したかを記録する。

実行ログは基本仕様には含めない。

### 12.1 記録する操作

| 操作 | 記録内容 |
|---|---|
| メガネに絵を入れる | どのメガネのどちら側に、何の絵を入れたか |
| メガネの絵を動かす | どこからどの座標に動かしたか |
| メガネの絵を消す | どの絵を消したか |
| メガネを消す | 消したメガネに入っていた絵 |

### 12.2 記録しない操作

次の操作は、基本的にはログに記録しない。

- メガネが実行された回数
- ループの回数
- 実行中に何個の絵が変化したか

## 13. サーバー構成

現在のシステムの構成を利用する。

| ファイル | 役割 |
|---|---|
| `server.js` | Expressサーバー |
| `public/index.html` | 画面 |
| `public/style.css` | 見た目 |
| `public/app.js` | フロントエンドの動作 |
| `public/images/` | 部品置き場に表示する画像 |
| `log.json` | ログ保存 |

## 14. API

現在のシステムを基本に、作品保存のAPIを追加できる構成にする。

| メソッド | パス | 内容 |
|---|---|---|
| GET | `/api/images` | 使用可能な画像一覧を取得する |
| GET | `/api/log` | ログ一覧を取得する |
| POST | `/api/log` | ログを追加する |
| GET | `/api/projects` | 保存済み作品一覧を取得する |
| POST | `/api/projects` | 作品を保存する |
| GET | `/api/projects/:id` | 作品を読み込む |
